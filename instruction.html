Инструкция по управлению сайтом i-FREE.

В данной инструкции описываются действия :

  • по присоединению файлов,
  • по добавлению, редактированию и удалению фотографий1.

I. Присоединение файлов.
    Пусть мы хотим разместить файл installer.rar, который должен быть доступен для скачивания с фотографии "NewProduct". Для этого нужно :

  1. Поместить файл в директорию, которая будет доступна через web. (Например, в директорию "/files/"2 проекта).
  2. Сформировать ссылку на размещённый файл, ("/files/installer.rar"),
  3. Открыть файл "polaroids.json"3, и найти описание нужной фотографии (в нашем примере, "NewProduct"),
  4.  Установить, свойству "downloadUri", ссылку полученную на шаге 2.

    
    Замечание. Если файл будет распологаться не в проекте, то шаг 1 может быть пропущен.
    
II. Добавление новой фотографии.
    Описание фотографий содержится в файле "polaroids.json"3. Также, изменения могут затронуть файлы "images.json", "polaroidgroups.json".
    

  1.  Для добавления новой фотографии, требуется
    •  две картинки этой фотографии,
    • текст-описание (опционально),
    • ссылка для скачивания\перехода (опционально),
  2. Для новой фотографии, создаётся запись в файле "polaroids.json".
    • обязательнами свойствами этой записи являются :
      • идентификатор новой фотографии - свойство "id" (уникальный среди всех фотографий) (нужен также, для связи с картиноками данной фотографии, см пункт II.4) .
      • тип фотографии - свойство "singleScreenType" (от него зависит отображение увеличенной версии фотографии). Возможные значения, перечислены в таблице 1 .
    • дополнительными свойствами являются :
      • текст описания (нужен, если тип фотографии - "single"). Свойство "text".
      • ссылка на скачивание\переход (для типа "tryit"). Свойство "downloadUri".
      • подпись под картинкой фотографии. Свойство "caption".
  3. Идентификатор новой фотографии нужно внести в одну из групп. Файл "polaroidgroups.json".
  4. Связать картинки, новой фотографии с фотографией.
    • подготовить по две картинки для каждого разрешения.
    • для каждого из разрешений, в папке "images/<имя разрешения>/photos" создать папку с названием, совпадающим с идентификатором новой фотографии. Имена разрешений, и соответствующие им разрешения привидены в таблице 2.
    • в файле "images.json" добавить записи о картинках фотографии. Записи следующего вида:

            {

                "id"            : "polaroid-<идентификатор новой фотографии>-big-photo",

                "path"            : "photos/<идентификатор новой фотографии>/<имя большой картинки>",

                "group"            : "popup"

            }

            и

            {

                "id"            : "polaroid-<идентификатор новой фотографии>-small-photo",

                "path"            : "photos/<идентификатор новой фотографии>/<имя малой картинки фотографии>",

                "group"            : "popup"

            }


III. Редактирование фотографии.
    Для редактирования фотографии нужно изменить её свойства, которые были описаны в п. II.
    
IV. Удаление фотографий.

  1. Для скрытия какой-либо фотографии, нужно удалить её идентификатор из её группы, в файле "polaroidgroups.json".
  2. Для полного удаления фотографии, нужно:
    • удалить идентификатор фотографии из её группы,
    • удалить запись о фотографии в файле "polaroids.json",
    • удалить записи о картинках фотографии из файла "images.json",
    • удалить картинки фотографии.
Приложения.

Таблица 1. Типы фотографий.
Тип фотографии Описание типа
"single" Используется для фотографий с текстом-описанием
"tryit" Используется для фотографий с ссылками перехода\скачивания
"comingsoon" Используется для фотографий, в описании которых - "The product is coming"

Примечания.

  1. В данной инструкции, под фотографией будет пониматься объект, объединяющий в себе :
    • 2 картинки,
    • подпись для картинки,
    • возможно текст описания,
    • возможно ссылку для перехода\скачивания.
  2. Все пути в инструкции начинаются с корневой директории проекта. Путь "/" означает корень проекта.
  3. Все файлы с расширением ".json" расположены в директории "/resources".
